# Architectural Decision Records (ADR)

This directory contains records of significant architectural and technical decisions made for the frontend of the Learnix platform.

We use these documents to keep track of *why* certain choices were made, the context around them, and the alternatives considered. 

## Decisions Index

- [API Layer & State Management](API.md)
- [Frontend Architecture](ARCHITECTURE.md)
- [Authentication & Role Management](AUTH.md)
- [Form Handling](FORMS.md)
- [Internationalization & SEO](I18N_SEO.md)
- [Linting & Formatting Strategy](LINTING_FORMATTING.md)
- [UI & Styling Approach](UI.md)

## Proposing a New Decision

When introducing a major change in tools, libraries, or architectural patterns:

1. Copy the `TEMPLATE.md` file in this directory.
2. Name it descriptively (e.g., `ROUTING.md`).
3. Fill out the sections following the template.
4. Submit a Pull Request to review the decision.
